# Этап A: индексатор кода

Этот файл фиксирует, что я уже реализовала на этапе A, как устроена база данных и как проверить результат руками.

## Что реализовано

На этапе A я собрала индексатор Python-кода, который:

- рекурсивно находит все файлы `*.py` в указанной директории;
- читает каждый файл как UTF-8;
- парсит код через стандартный модуль `ast`;
- извлекает:
  - `class`-определения;
  - `def`-функции;
  - `async def`-функции;
  - методы классов;
  - вложенные функции;
- сохраняет результаты в SQLite;
- не падает на битом файле, а пропускает его с предупреждением;
- умеет пересоздавать таблицы перед новой индексацией, чтобы запуск был идемпотентным;
- поддерживает запуск из CLI.

Основная логика находится здесь:

- [app/indexer.py](app/indexer.py)
- [app/models.py](app/models.py)
- [app/db.py](app/db.py)

## Где находится база данных

По умолчанию индексатор создаёт SQLite-базу в корне проекта:

- `code_index.db`

То есть при запуске без параметра `--db` база появится прямо рядом с `app/`, `dataset/` и `STAGE_A.md`.

Если указать свой путь через `--db`, база будет создана там, куда вы укажете файл.

Для тестового мини-проекта, который лежит в `dataset/`, база создаётся по тому пути, который я передаю в `--db`. Обычно это:

- `dataset/code_index.db`

## Таблицы в БД

### `files`

Хранит проиндексированные файлы.

Поля:

- `id` - первичный ключ;
- `name` - имя файла;
- `path` - относительный путь до файла;
- `definitions` - связь с таблицей `definitions`.

### `definitions`

Хранит найденные классы и функции.

Поля:

- `id` - первичный ключ;
- `file_id` - ссылка на файл;
- `name` - имя функции или класса;
- `kind` - тип сущности: `function` или `class`;
- `line_start` - первая строка определения;
- `line_end` - последняя строка определения;
- `docstring` - docstring, если он есть.

## Индексы и ограничения

В модели уже заложены полезные ограничения и индексы:

- `files.name` - `unique` и `index=True`;
- `definitions.file_id` - `index=True`;
- `definitions.name` - `index=True`.

Это удобно для поиска по имени файла, соединения таблиц и поиска по названию определения.

## Что можно получить из БД

Из этой базы можно быстро посмотреть:

- какие файлы были найдены;
- какие классы и функции есть в каждом файле;
- в каких строках начинается и заканчивается определение;
- есть ли у него `docstring`;
- сколько в проекте всего файлов и определений.

## Полезные SQL-запросы

### Посмотреть все файлы

```sql
select * from files;
```

### Посмотреть все определения

```sql
select * from definitions order by file_id, line_start;
```

### Посмотреть структуру файла вместе с именем файла

```sql
select
  f.name as file_name,
  f.path,
  d.name,
  d.kind,
  d.line_start,
  d.line_end,
  d.docstring
from definitions d
join files f on f.id = d.file_id
order by f.id, d.line_start;
```

### Посмотреть только функции

```sql
select * from definitions where kind = 'function';
```

### Посмотреть только классы

```sql
select * from definitions where kind = 'class';
```

### Посчитать файлы и определения

```sql
select count(*) as files_count from files;
select count(*) as definitions_count from definitions;
```

### Найти определения по имени

```sql
select * from definitions where lower(name) like lower('%auth%');
```

### Найти определения с docstring

```sql
select * from definitions where docstring is not null;
```

## Как запустить индексатор

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Запустить индексатор на папке с кодом

По умолчанию индексатор ищет файлы в папке `dataset/`:

```bash
python3 -m app.indexer dataset/
```

Если нужен свой путь к базе:

```bash
python3 -m app.indexer dataset/ --db code_index.db
```

Если база должна лежать в подпапке, она создастся автоматически.

### 3. Проверить результат

После запуска можно открыть `code_index.db` в `DBCode` или выполнить SQL-запросы через `sqlite3`, `DB Browser for SQLite` или любой другой клиент.

## Как проверить на тестовой репе

Для проверки парсера я собрала отдельный мини-проект в `dataset/`:

- [dataset](dataset)

Там есть:

- несколько модулей;
- одинаковые имена файлов в разных папках;
- классы, методы, async-функции;
- вложенные функции;
- битый `.py`-файл.

Запуск на этой репе:

```bash
python3 -m app.indexer dataset/ --db dataset/code_index.db
```
